Example run

This page shows an example run with a configuration of four players: one villagers, two mafiosi, and one detective. This configuration was chosen as it is the minimal configuration that incorporates both the extra knowledge of mafia and the special ability of the detective. It thus provides the clearest overview that also incorporates all rules of the model (described in Implementation), albeit that the townfolk is not the majority. To keep this run as simple as possible, only two talking rounds are allowed per discussion phase. The following subsections will walk through the example run phase by phase.

Day 1



❮ ❯



Figure 2 illustrates the initial phase of the example run. It is evident that the model is highly complex right from the beginning, and all agents believe a large amount worlds to be possible. The actual world, denoted as m*vdm*, consists of agent 1 and 4 being mafia, agent 2 being a villager, and agent 3 acting as the detective. In this case, only the mafia members are flagged as suspicious, indicated by the asterisk (*) appended to their names. During the discussion phase, agent 4 initiates a conversation with agent 2, and afterwards, agent 2 engages in dialogue with agent 4. Consequently, agent 2 becomes aware of agent 4's suspicious behavior by (agent2 talks to agent4 ∧ sus4) → K2sus4. As a result, R2 limits the possible worlds to only those accommodating this new information. Note that if these conversations had happened the other way around, that the mafia would have found out about the suspicion of agent 2.

Night 1


❮ ❯



Figure 3 illustrates the first night phase of the example run. During this phase, the detective acquires crucial information, discovering that agent 4 is a member of the Mafia. This discovery triggers the logical rule (d3 ∧ inspect agent4 ∧ m4) → K3m4, which comes into effect. As shown in Figure 3.2, the detective's belief now restricts possible worlds to those where agent 4 is identified as a mafia member.
Unfortunately for the townfolk, the mafia collectively decides to eliminate the detective. Consequently, the logical rule dead3 → C dead3 takes effect. This leads to a narrowing of possible worlds, where only scenarios in which the detective is agent 3 remain viable for the agents, as depicted in Figure 3.4.

Day 2



❮ ❯



There are only three players left now, two of which are mafia. The townfolk are thus already certain of defeat. The model, however, runs until only mafia or townfolk are left, so we proceed to the next day phase. Only the first talking round is used by the agents, where agents 4 and 2 exchange knowledge. Agent 2 learns nothing new, as he/she already knew agent 4 to be suspicious. Agent 4 however, now finds out that agent 2 is suspicious of him, because of (agent4 talks to agent2 ∧ K2sus) → K4K2sus4. The mafia then decides to vote out agent 2, leading to their victory since there are no townfolk remaining. In this case, agent 2 is chosen for elimination because the mafia never vote against each other. However, if there were more agents remaining, the same outcome would occur based on their voting/murder rule: K4K2sus4 ∧ ¬ dead2.
